Machine access

Use Context Stack through MCP

Context Stack is the framework. MCP is the access path that lets AI assistants consult it during delivery and assessment.

https://mcp.context-stack.org/mcp

What it does

  • Exposes the stack overview, glossary, and locked decisions.
  • Routes users to ContextOps, ContextBoundary, Sthala, or Griha.
  • Supports framework discovery, assessment prompts, and delivery guidance.
  • Lets AI assistants ask which governance layer applies before work proceeds.

What it does not do

  • It does not enforce policy.
  • It does not store prompts or assessment answers.
  • It does not replace gateways, IAM, policy engines, egress controls, or runtime controls.
  • It does not make compliance claims by itself.

Example questions

  • Which part of Context Stack applies to stale runbooks after AMS handover?
  • Is this a ContextOps issue, a ContextBoundary issue, or a Sthala issue?
  • What should we assess first for AI context governance?
  • Which governance questions should we ask before an AI workflow calls an external API?
  • Which framework document should a delivery team read first?

Assessment use

Use the MCP endpoint as a framework consultation layer. An AI assistant can ask which stack layer applies, retrieve the right source, and guide a first-pass assessment of context ownership, egress boundaries, runtime placement, and approval paths.

Enforcement remains deterministic infrastructure work. Agents learn the rules through MCP; gateways, IAM, OPA, egress controls, approval workflows, and runtime controls enforce them.
