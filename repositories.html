Context Stack Repositories

Canonical repository map for the open-source Context Stack.

The stack is organized around a simple premise: probabilistic intelligence becomes production-ready only when it is placed inside deterministic governance boundaries, explicit context handling, and operational controls.

Core Repositories

  • ContextOps Governance-responsibility layer for AI context.

    ContextOps defines ownership, accountability, trust classification, lifecycle, governance metadata, and context source accountability. It says how an organisation governs the context AI systems depend on.

  • ContextBoundary Deployment-agnostic egress governance specification.

    ContextBoundary defines where AI-processed data is permitted to flow. It governs Egress Tier controls, capability visibility, MCP filtering, invocation control, approval workflows, egress control, and auditing.

  • Sthala Structural layer for governed AI execution spaces.

    Sthala represents the place or structure in which governed AI activity occurs. It frames how components, contexts, and boundaries relate to each other inside an AI governance architecture.

  • Griha Application layer for usable governed AI systems.

    Griha is the layer where governed AI capabilities become usable products or workflows. It connects the underlying governance architecture to practical user-facing systems.

  • context-stack Integrated stack tying the governance components together.

    context-stack is the canonical coordination layer for locked terminology, cross-project decisions, and family framing across ContextOps, ContextBoundary, Sthala, and Griha.

  • context-stack-mcp Machine interface for the stack.

    context-stack-mcp exposes read-only MCP resources, prompts, and guidance tools so AI clients can discover the stack and route users to the correct framework layer.
