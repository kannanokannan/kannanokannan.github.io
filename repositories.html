Repository map

Context Stack Repositories

The stack is split by governance responsibility. Use the assessment on the homepage when you are unsure where to start.

This page is the canonical map for people who already know they need the source repositories.

Core repositories

  • ContextOps Governance-responsibility layer for AI context.

    Defines ownership, accountability, trust classification, lifecycle, governance metadata, maturity, and context source accountability.

  • ContextBoundary Deployment-agnostic egress governance specification.

    Defines where AI-processed data is permitted to flow through Egress Tier controls, vendor zones, approval workflows, and audit profiles.

  • Sthala Governed runtime reference under ContextBoundary.

    Frames AI runtime placement under approved boundaries, with deterministic execution constraints and explicit egress control.

  • Griha Product and adoption layer.

    Composes the governance architecture into usable product and workflow surfaces.

  • context-stack Canonical coordination repository.

    Holds locked terminology, cross-project decisions, and family framing across the stack.

  • context-stack-mcp Read-only machine interface.

    Exposes Context Stack resources, prompts, and guidance tools through MCP so AI clients can route users to the correct layer.
